# Target Users

AegisDesk is designed for organizations that want to use AI in cloud operations without turning AI into an unmanaged shadow IT risk.

## Primary Personas

### 1. Cloud Operations Engineer

Needs:

- Triage incidents faster
- Summarize logs and alerts
- Create or update tickets
- Search internal runbooks
- Avoid leaking secrets or customer data

Value:

- Faster first response during common cloud incidents
- Safer handling of logs and operational context
- Less manual ticket and runbook work

### 2. Platform Engineer

Needs:

- Provide self-service workflows to application teams
- Enforce platform policies consistently
- Standardize how AI calls tools
- Reduce repetitive support requests

Value:

- AI becomes part of the internal platform instead of a separate uncontrolled tool
- Policies are encoded once and reused across workflows
- Tool execution is auditable and scoped

### 3. Engineering Manager

Needs:

- Give teams useful AI assistance
- Keep risky actions behind approval
- See how AI is being used
- Explain controls to leadership

Value:

- Better team productivity without losing oversight
- Clear approval queue for sensitive actions
- Dashboard evidence for AI governance

### 4. Security or Compliance Reviewer

Needs:

- Know whether sensitive data is being exposed
- Review policy decisions
- Audit tool calls and approvals
- Confirm least-privilege behavior

Value:

- Evidence of redaction, denial, approval, and routing decisions
- Repeatable policy rules instead of informal guidance
- Better control over AI use in regulated workflows

### 5. FinOps Practitioner

Needs:

- Understand AI usage and cost
- Attribute spend to teams or use cases
- Detect budget risk early
- Compare local and cloud model routing decisions

Value:

- AI spend visibility at request level
- Budget-aware routing
- Clear link between AI consumption and business workflow

## Demo Personas

### Employee

Uses chat to ask for help, request access, summarize logs, and create tickets.

### Manager

Approves or rejects scoped actions such as temporary read-only access.

### Admin

Reviews model usage, cost, redactions, policy decisions, tool calls, and evaluation results.
